## Project Description

AdMotion is a Digital-Out-of-Home (DOOH) advertising platform that transforms urban vehicles into intelligent advertising displays using roof-mounted LED panels. The system consists of:
- **Web Admin Dashboard** (React.js): Fleet and campaign management
- **Vehicle Display App** (PWA): Real-time ad rendering on Android TV boxes
- **Backend API** (FastAPI/Python): AI scheduling and analytics

The platform enables fleet owners to monetize vehicles (30-50% revenue increase) while providing advertisers with cost-effective, real-time advertising (60-70% cost savings vs traditional DOOH).

---

## Objectives

- Develop web-based admin dashboard with complete fleet and campaign management
- Create vehicle display PWA with real-time synchronization (<200ms latency)
- Implement AI-powered ad scheduling and distribution engine
- Build secure, scalable Firebase infrastructure with 99.9% uptime
- Deliver comprehensive documentation (BRD, SRS, API docs, user guides)

---

## Project Scope

### Deliverables

**Software Applications:**
- Admin Dashboard (React.js): User management, vehicle registration, campaign management, analytics
- Vehicle Display App (PWA): Real-time sync, multi-layout display, offline operation, heartbeat monitoring
- Backend API (FastAPI): AI scheduling, synchronization, analytics, payment processing

**Documentation:**
- Business Requirements Document (BRD)
- Software Requirements Specification (SRS)
- User Stories & Use Cases
- System Architecture & Database Schema
- API Documentation & User Guides
- Testing & Deployment Documentation

**Technology Stack:**
- Firebase (Firestore, Authentication)
- React.js + Tailwind CSS
- FastAPI + Python
- Vercel deployment

### Out of Scope
- Hardware manufacturing and LED production
- Physical vehicle installation
- GPS hardware development
- Native mobile apps (Phase 2)
- International expansion (Phase 2)

## Risks / Issues

**Technical Risks:**
- **Real-Time Sync Failure**: Redundant connections, offline queuing, retry mechanisms
- **Security Vulnerabilities**: Security audits, encryption, incident response plan
- **Performance at Scale**: Load testing, horizontal scaling, caching strategies

**Project Risks:**
- **Scope Creep**: Clear requirements, change control, feature prioritization
- **Timeline Delays**: Agile sprints, buffer time, MVP focus
- **Resource Constraints**: Skill development, external support if needed
- **Integration Issues**: Early testing, API mocks, incremental integration
